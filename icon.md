## Usage Context

### File Type Icons
These icons are used to represent different file types in the file explorer and editor tabs. The system automatically selects the appropriate icon based on the file extension. **[BRAND: All icons were originally branded as Void and need to be updated to Prism branding]**

### Application Icons
These icons are used throughout the application as the main brand representation in:
- Application window title bars
- Taskbar/Dock icons
- Installation packages
- Desktop shortcuts
- File associations

**[BRAND: All application icons need to be updated from Void to Prism branding]**

### Installation Icons
These icons are used during the installation process to provide visual branding and guidance to users. **[BRAND: All installation icons need to be updated from Void to Prism branding, except inno-prism.bmp which is already updated]**

## Icon Sizing Standards

### Windows
- Various ICO formats supporting multiple resolutions
- PNG icons for modern Windows features
- BMP files for installer wizard backgrounds

### macOS
- ICNS format supporting multiple resolutions in a single file

### Linux
- PNG format for desktop integration
- Desktop entry files for system integration

## File Associations

The installer associates these icons with specific file types to provide visual recognition in the operating system's file explorer. **[BRAND: All file associations were originally branded as Void and need to be updated to Prism branding]**

## Branding Transition Notes

### Icons Already Updated to Prism Branding

### Icons Needing Brand Transition from Void to Prism
All other icons in the application were originally branded as "Void" and need to be updated to "Prism" branding to complete the rebranding process. This includes:
- All application icons (code.ico, code.icns, code.png, etc.)
- All file type icons (c.ico, cpp.ico, javascript.ico, etc.)
- All installer wizard icons (inno-big-*.bmp, inno-small-*.bmp, etc.)
- All server icons (code-192.png, code-512.png, favicon.ico)
- All desktop integration files (code.Desktop, code.appdata.xml, etc.)
- All logo assets (logo_cube_noshadow.png)

## Branding Consistency

All icons should maintain visual consistency with the Prism Editor brand identity while providing clear visual distinction for different file types and application contexts. The transition from Void to Prism branding should be completed across all icon assets to ensure consistent user experience.
